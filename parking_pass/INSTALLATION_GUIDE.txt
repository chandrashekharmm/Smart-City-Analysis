## 📋 QUICK INSTALLATION (5 Minutes)

### Step 1: Extract Files
1. Extract `parking_system_COMPLETE.zip`
2. Copy the `parking_system` folder to: `C:\xampp\htdocs\`
3. Final path should be: `C:\xampp\htdocs\parking_system\`

### Step 2: Create Database
1. Start XAMPP (Apache + MySQL)
2. Open: http://localhost/phpmyadmin
3. Click "New" to create database
4. Database name: `parking_system`
5. Click "Create"

### Step 3: Import Database
1. Click on `parking_system` database (left sidebar)
2. Click "Import" tab (top menu)
3. Click "Choose File"
4. Select: `C:\xampp\htdocs\parking_system\database.sql`
5. Click "Go" at the bottom
6. Wait for success message

### Step 4: Access the System
**Admin Panel:**
- URL: http://localhost/parking_system/admin/login.php
- Username: `admin`
- Password: `admin123`

**User Panel:**
- URL: http://localhost/parking_system/
- Create new user account or use existing

## 📧 ENABLE EMAILS (Optional)

Only if you want to send booking confirmation emails:

### For Gmail:

1. **Get App Password:**
   - Go to: https://myaccount.google.com/security
   - Enable 2-Factor Authentication
   - Go to: https://myaccount.google.com/apppasswords
   - Generate app password for "Mail"
   - Copy the 16-character password

2. **Edit `email_config.php`:**
   ```php
   define('ENABLE_EMAIL', true);  // Change to true
   define('SMTP_USERNAME', 'youremail@gmail.com');
   define('SMTP_PASSWORD', 'your-16-char-app-password');
   ```

3. **Save and test** by creating a booking

## ❓ TROUBLESHOOTING

### Issue: Can't login to admin panel
**Solution:**
- Username: `admin` (lowercase)
- Password: `admin123`
- Or run: `admin/reset_password.php` to reset

### Issue: Database connection error
**Solution:**
1. Check XAMPP MySQL is running
2. Verify database name is `parking_system`
3. Check `config.php` settings

### Issue: Blank page after clicking
**Solution:**
1. Check Apache error logs: `C:\xampp\apache\logs\error.log`
2. Enable PHP errors in `php.ini`:
   ```
   display_errors = On
   error_reporting = E_ALL
   ```

### Issue: Email errors (even though emails disabled)
**Solution:**
- Make sure you replaced `email_config.php` with the new version
- Verify line 13 says: `define('ENABLE_EMAIL', false);`
